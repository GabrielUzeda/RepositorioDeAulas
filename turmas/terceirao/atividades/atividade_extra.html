Atividade Avaliativa Integrada

Projeto Gerador de Questões ENEM

Instruções: Esta atividade tem como objetivo avaliar a compreensão conceitual sobre o funcionamento do sistema que desenvolvemos. Responda às questões de forma discursiva, utilizando suas próprias palavras e as analogias discutidas em aula.
    Bloco 1: Arquitetura Geral (O Restaurante Digital)

  1. Questão 1 — Analogia do Restaurante

    No contexto do nosso projeto, utilizamos a analogia de um restaurante para explicar o funcionamento de uma API. Explique quem representa o Cliente, o Garçom e a Cozinha nesta analogia e qual a função de cada um no sistema de geração de questões.

  2. Questão 2 — Formato JSON

    O formato JSON é fundamental para a comunicação entre as equipes (Python, IA, Banco de Dados). Por que não podemos simplesmente enviar os dados em português ou inglês simples? Qual a vantagem de usar uma estrutura rígida como o JSON?

  3. Questão 3 — Python vs SQL

    Diferencie o papel da Equipe Python e da Equipe SQL no projeto. Se o Python é o "cérebro" que conecta tudo, por que precisamos de um sistema separado (SQL) apenas para guardar os dados?

  4. Questão 4 — Rotas no Flask

    Explique o que é uma Rota (ex: @app.route('/questoes')) no Flask. Fazendo um paralelo com a navegação na internet ou o endereço de uma casa, para que ela serve?

  5. Questão 5 — Comando cURL

    O que é o comando curl e por que ele foi utilizado durante o desenvolvimento antes de termos um site com botões e cores (Frontend)?

    6. Bloco 2: Banco de Dados (A Biblioteca)

  6. Questão 6 — Modelagem de Dados

    Ao modelar o banco de dados, decidimos separar os dados em duas tabelas principais: questoes e alternativas. Por que não colocar as 5 alternativas na mesma tabela da questão (ex: colunas alternativa_A, alternativa_B, etc.)? Cite uma desvantagem dessa abordagem de "tabela única".

  7. Questão 7 — Primary Key

    Explique o conceito de Primary Key (Chave Primária). Por que é arriscado identificar uma questão apenas pelo seu "enunciado" em vez de usar um número de ID único?

  8. Questão 8 — Foreign Key

    O que é uma Foreign Key (Chave Estrangeira) e qual a função dela na tabela de alternativas? O que aconteceria se tentássemos criar uma alternativa sem essa chave?

  9. Questão 9 — Restrição NOT NULL

    No comando SQL, usamos a restrição NOT NULL em campos como "enunciado". Qual é o impacto dessa regra para a qualidade dos dados armazenados no sistema?

  10. Questão 10 — Tipo BOOLEAN

    O tipo de dado BOOLEAN foi usado para indicar se uma alternativa é correta ou não. Como o computador interpreta esse tipo de dado e quais são os únicos valores possíveis para ele?

  11. Questão 11 — Integridade Referencial

    O que significa a regra ON DELETE CASCADE configurada na relação entre questões e alternativas? Por que ela é importante para manter a "limpeza" do banco de dados?

    12. Bloco 3: IA e Engenharia de Prompt

  12. Questão 12 — Personas

    Na engenharia de prompts, discutimos o conceito de Persona. Como definir uma persona (ex: "Aja como um professor de História") altera a resposta gerada pela IA em comparação a um pedido genérico?

  13. Questão 13 — Alucinação da IA

    O que é uma alucinação em Inteligência Artificial? Dê um exemplo de como isso poderia prejudicar um estudante que estivesse usando nosso gerador de questões.

  14. Questão 14 — Qualidade do Prompt

    Por que um prompt vago como "Crie uma questão sobre o Brasil" é considerado ruim? Liste três elementos que deveriam ser adicionados a esse pedido para torná-lo um prompt eficaz.

  15. Questão 15 — Estratégia de Dupla Camada

    Explicamos a estratégia de "Dupla Camada" (Geração e Verificação). Por que confiar na primeira resposta da IA pode ser perigoso, e como a segunda camada ajuda a filtrar conteúdos impróprios ou ofensivos?

  16. Questão 16 — Funcionamento dos LLMs

    A IA não "sabe" as respostas como um humano sabe. Se ela é apenas um modelo de previsão de palavras, como ela consegue montar uma questão de física corretamente? (Explique simplificadamente o conceito de treinamento/padrões).

    17. Bloco 4: Segurança e Backend

  17. Questão 17 — SQL Injection

    O que é SQL Injection? Explique como um usuário mal-intencionado poderia usar uma caixa de texto do site para roubar ou apagar dados se o sistema não for protegido.

  18. Questão 18 — Autenticação vs Autorização

    Qual a diferença entre Autenticação e Autorização? Use o exemplo de um festival ou show para ilustrar a diferença.

  19. Questão 19 — Token JWT

    O Token JWT foi comparado a um "crachá VIP". Se um usuário consegue forjar (falsificar) esse crachá, o que ele conseguiria fazer no sistema?

  20. Questão 20 — Estrutura do JWT

    O JWT possui três partes: Header, Payload e Signature. Por que dizemos que não devemos guardar senhas no Payload, mesmo que o token pareça um monte de letras aleatórias?

  21. Questão 21 — Segurança de Credenciais

    Por que é considerado um erro grave de segurança escrever a senha do banco de dados diretamente no código do programa (hardcoded)? Qual é a solução correta para armazenar essas senhas?

  22. Questão 22 — Ngrok

    O que é o ngrok e por que precisamos dele para conectar a IA que está rodando no nosso computador da escola com o servidor que está na nuvem (internet)?

    23. Bloco 5: Lógica de Programação

  23. Questão 23 — Depuração de JSON

    Analise o seguinte trecho de JSON. Identifique um erro de sintaxe nele e explique por que ele impediria o sistema de ler os dados:

    {
  "materia": "Matemática",
  "dificuldade": "Difícil",
  "tags": ["geometria", "calculo",]
}

  24. Questão 24 — Requisições Síncronas

    No Python, usamos a biblioteca requests para "falar" com a IA. Se a IA demorar 30 segundos para responder, o que acontece com o nosso código Python enquanto isso? (Conceito de requisição síncrona/espera).

  25. Questão 25 — Métodos HTTP

    O método HTTP GET serve para buscar dados, enquanto o POST serve para enviar. Se quiséssemos criar uma funcionalidade para "Atualizar uma questão errada", qual método seria o mais adequado conceitualmente (além de GET e POST)?

  26. Questão 26 — Códigos de Status HTTP

    Explique a diferença entre um Erro de Cliente (4xx) e um Erro de Servidor (5xx). Se o usuário tentar acessar uma questão que não existe (ID 9999), qual tipo de erro o sistema deve retornar?

  27. Questão 27 — Listas vs Objetos

    Por que usamos listas (arrays []) dentro do JSON para as alternativas, mas objetos ({}) para os dados da questão? Qual a característica das listas que as torna ideais para as alternativas?

  28. Questão 28 — Padronização e Acoplamento

    Se a equipe de Banco de Dados mudar o nome da coluna enunciado para texto_da_pergunta, o que acontecerá com o código da Equipe Python e da Equipe JSON? O que isso nos ensina sobre padronização?

  29. Questão 29 — Gerenciamento de Pacotes

    Qual a função do arquivo requirements.txt ou do comando pip install em um projeto Python? Por que não podemos apenas copiar o arquivo .py para outro computador e esperar que funcione?

  30. Questão 30 — Síntese do Sistema

    Descreva, em um parágrafo, o caminho completo que uma nova questão percorre desde o momento em que o professor digita o tema no site até o momento em que ela é salva no banco de dados.

    Frontend → API → IA → API → Banco de Dados
